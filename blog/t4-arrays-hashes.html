Storing items: junk vs labeled | Oct 5, 2014

For thinking about the differences between arrays and hashes in Ruby, let's think about people's garages. Specifically how people store things in their garage. Because that's what arrays and hashes are in Ruby and in other Programming languages. It is how you store the crap.

Everybody has a junk box where there is no labeling to what should be in the box. Anything can be thrown into that box, there is no type, whether it is coins, photos, cd, food, or whatever. I like to think of arrays as that. It is a mix bag storage type where you can throw numbers, words or even other arrays (storage boxes) in there! To me, arrays are like junk boxes. There will be a name outside the junk box but anything can be thrown and stored in that box.

Hashes provide for more ordering and labeling. You see anything stored in a Hash box has an accompanying key and a value. Let's think of this as a box storing cds or dvds (yes I know that most people don't own these anymore). When you place an cd or dvd, on the outside is the name and what type of entertainment it is. You can tell if it is a Disney movie or a Linkin Park CD because it is clearly labeled. To make for easier access, when you store something in a Hash, it must have a label of key and value. When we throw some clothes in the Hash gargage box, we will need to label it as 'shirtcolor' => 'blue'. That is how you label in a Hash, using 'key' => 'value'. Storing a DVD would be 'fav_movie' => 'Ratatouille' and so on.

The great thing about both of these storages is that are indexed as you store the items in the box. Index is just the position and order of where the item is stored. A weird thing we must keep in mind with these storages is that it starts with a zero index. Which is just a fancy way of saying the first item stored is the zero index. So if you want to retrieve the first item you placed in the junk box, you can just look for junkbox[0]. If you want to get the sixth items you placed, you will look for junkbox[5]That is the term for arrays in Ruby, the storagename[indexposition] of the item you are looking for. Because Hashes have a specific key and value, you can search for them in the storage box much faster. Don't remember the index number, just search for their key, e.g. hashbox["fav_movie"]
