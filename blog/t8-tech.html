Recursion ... | Nov 9, 2014

When I ask the Developers I use to work with about recursion, the always come back with this joke: "In order to understand recursion, you need to understand recursion." Well I don't understand recursion but this blog will be about researching on this fascinating Computer Science topic. The most basic definition of recursion is a function that calls itself.

Usually in a program, a function is declared and defined at the beginning of a file. Please see the simple function below.

def print_name(name)
print "Hello, #{name} how's it going? Have a good day, buddy."
end

In order to get the function to work, we need to call it using the function name of:

print_name("Leonard)

Recursion breaks down a large, complex problem into smaller versions of itself. This post on stackoverflow gave a pretty good explanation on recursion with the following example:

How do you empty a vase containing three flowers? Answer: if the vase is not empty, you take out one flower and then you empty a vase containing two flowers.

How do you empty a vase containing two flowers? Answer: if the vase is not empty, you take out one flower and then you empty a vase containing one flower.

How do you empty a vase containing one flower? Answer: if the vase is not empty, you take out one flower and then you empty a vase containing no flowers.

How do you empty a vase containing no flowers? Answer: if the vase is not empty, you take out one flower but the vase is empty so you're done.

That's repetitive. Let's generalize it:

How do you empty a vase containing N flowers? Answer: if the vase is not empty, you take out one flower and then you empty a vase containing N-1 flowers.

This is just a general simple explanation. In fact, with that example we could have used a for loop. But with more complex problems where you can break it down to smaller, recursive steps, a recursive function would be better. When at Devbootcamp, I hope to learn more and understand recursion so that I can now make the joke to others who ask me about it!
